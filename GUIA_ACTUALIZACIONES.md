# 🔄 Guía de Actualizaciones del Sistema

## 🎯 3 formas de actualizar sin perder datos

### 🥇 **Opción 1: GitHub (Automática)**
**✅ La más profesional - Una vez configurada, es automática**

**Configuración inicial (una sola vez):**
1. Sube tu proyecto a GitHub
2. Tu cliente ejecuta: `Actualizar_Sistema.bat`
3. Ingresa la URL de tu repositorio
4. ¡Listo! Ya está configurado

**Para futuras actualizaciones:**
1. Tú haces cambios y los subes a GitHub
2. Tu cliente ejecuta: `Actualizar_Sistema.bat`
3. Automáticamente se descargan los cambios
4. Sus datos se mantienen intactos

---

### 🥈 **Opción 2: Manual (Simple)**
**✅ Sin configuración técnica**

**Cuando hagas cambios:**
1. Le envías solo los archivos que cambiaste (ZIP pequeño)
2. Tu cliente ejecuta: `Actualizar_Manual.bat`
3. Sigue las instrucciones para reemplazar archivos
4. Sus datos se mantienen seguros

---

### 🥉 **Opción 3: Notificación web**
**✅ Se entera automáticamente que hay actualizaciones**

En el sistema aparece una notificación cuando hay nueva versión:
- "¡Nueva actualización disponible!"
- Botón para solicitar la actualización por email
- No interrumpe su trabajo

## 🛠️ Proceso técnico (para ti)

### Con GitHub:
```bash
# 1. Haces cambios en tu código
git add .
git commit -m "Mejora: nueva funcionalidad X"
git push origin main

# 2. Tu cliente ejecuta Actualizar_Sistema.bat
# 3. ¡Automático!
```

### Manual:
```bash
# 1. Preparas solo los archivos modificados
# 2. Los envías en ZIP pequeño
# 3. Cliente ejecuta Actualizar_Manual.bat
# 4. Reemplaza archivos manualmente
```

---

## 💡 Recomendación

**Para clientes no técnicos:** Usa la **Opción 2 (Manual)**
- Más simple de explicar
- Sin configuración de Git
- Control total sobre qué se actualiza

**Para clientes técnicos:** Usa la **Opción 1 (GitHub)**
- Actualización con un clic
- Historial de cambios
- Más profesional
